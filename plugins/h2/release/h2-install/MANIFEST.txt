# harness-helm 설치 manifest
# 규칙: 이 파일에 작성하는 주석은 한글로 기입한다.
# 형식: <kind> <source> <destination>
#   kind:        copy | copy_dir | managed
#   source:      h2-install/ 기준 상대 경로(이 manifest가 있는 디렉터리)
#   destination: --target root 기준 상대 경로
# 참고:
#   - copy_dir source는 '/'로 끝나야 한다.
#   - 중복 destination과 '..' segment는 거부된다.
#   - docs/_archive/는 `.gitkeep`만 포함한 clean scaffold로 seed하고 source history는 제외한다.
#   - docs/_indexes/는 clean generated index 파일만 seed한다.
#     target project는 `harness kb-index`로 index 내용을 재생성한다.
#   - docs/01_plan ~ docs/04_report는 README.md만 seed한다.
#     workflow artifact는 target project가 생성한다.
#   - docs/10_domain ~ docs/50_operations는 source-only 제외 규칙을 거쳐 하위 tree를 seed한다.
#   - harness-helm runtime guideline은 docs/40_knowledge/conventions/guidelines/harness-helm/ 아래에 둔다.
#     기존 flat h2-*.md guideline은 canonical target path가 아니다.
#   - docs/40_knowledge/learnings/는 README.md scaffold만 target payload에 남긴다.
#     source repository learning 본문은 target project에 적용하지 않는다.
#   - docs/50_operations/releases/*.release-notes.md는 payload에서 제외한다.
#     source repository release notes는 target project에 적용하지 않는다.
